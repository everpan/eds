Overview

EDS API is a set of C++ header files and libraries which allows connecting to EDS servers from external software.

All libraries provided in this SDK support Win32 and Linux and work with all versions of EDS from 7.3 upwards.

In order to support a wide range of EDS versions without having to constantly recompile and link software, EDS API has been split into two layers:

  • Frontend library - version independent loader library
  • Backend libraries - version dependent connector libraries

Frontend library

The EDS API frontend library is shipped in source code form in edsapi/include and edsapi/src directories. These files form a shim layer which automatically handles loading of appropriate backend libraries and hides differences in their APIs.

To use the frontend library, include all files from the edsapi/src directory in your project (i.e. by compiling them and linking into an executable / library).

If compiling the frontend with Visual C++, please disable Unicode support and make sure that C++ exceptions are enabled (using /EHsc or /EHa compiler switches).

Please refer to provided API reference for more information on the interface of the frontend library.

Backend libraries

The EDS API backend libraries are shipped in binary form in edsapi/backend subdirectories. Each of these directories contains one or more dynamic libraries (.dll files on Windows, .so files on Linux) which implement client protocol of a specific EDS version.

In order to connect from your application to a specific version of EDS server, all library files from appropriate edsapi/backend subdirectory need to be shipped as part of that application.

The EDS API frontend will try to dynamically load these backend libraries at runtime. Please make sure, that backend libraries are placed in a directory which is searched by the dynamic library loader.

The recommended locations for backend libraries are:

  • the directory containing the main executable (.exe) of your application on Windows
  • a directory listed in the LD_LIBRARY_PATH environment variable on Linux platforms
Warning
Please do not statically link your application with these libraries! The list of symbols exported from these files may change between EDS versions. The only supported method of linking with backend libraries is through the Frontend library.

Getting started

This section outlines some basic steps required to use EDS API in a project:

  1. Import all .cpp files from the edsapi/frontend into the project (they should be compiled and linked with the rest of project code).
  2. Add the edsapi/include directory to the list of include file paths (use /I compiler switch for Microsoft Visual C++ and -I for GCC).
  3. Decide which versions of EDS need to be supported by your application and copy all library files from appropriate edsapi/backend subdirectories to a path within your project. These files need to be shipped with your application.
    Make sure that the location of these libraries is searched by the dynamic library loader (LoadLibrary on Windows, dlopen on Linux). This may require additional steps on some operating systems (like adding that directory to the LD_LIBRARY_PATH environment variable on Linux).
  4. Write code using classes from the Frontend library to access data on the EDS server.
    For more information, please refer to the API reference and examples from the edsapi/examples directory.

Requirements

EDS API frontend library is designed to work with any C++ 2003 standard conforming compiler.

EDS API backend libraries are built for specific versions of operating system. Please always use backend libraries from appropriate EDS API distribution package for a given platform (OS version).

The following operating systems and compilers have been tested and are known to work correctly:

  • Windows:
    • Microsoft Visual C++ 2008
  • Red Hat Enterprise Linux 5:
    • GCC 4.1
  • Red Hat Enterprise Linux 6:
    • GCC 4.4
