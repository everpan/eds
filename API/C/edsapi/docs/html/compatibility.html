Supporting multiple EDS versions

EDS API is designed to allow connecting to multiple versions of EDS servers without the need to recompile any code.

However, in order to support a wide range of EDS versions, some precautions need to be taken when integrating EDS API into application:

  • EDS API must always be used through the Frontend library layer. Do not statically link your application to any of the backend libraries.

  • Future releases of EDS may extend API with new methods. These methods will not be available in backend libraries for older EDS versions.

    When attempting to call such method on an old backend, the frontend library will throw an eds::UnsupportedFunctionError exception.

    Note that descriptions of all eds::live::LiveClient and eds::arch::ArchClient methods specify which version of EDS introduced that method.

  • Do not load different versions of the same backend at the same time (e.g. LiveClient version 9.0 and 9.1) in a single process.

    Different versions of backend libraries may internally export the same subset of symbols which could confuse dynamic linker.

    Creating several clients with the same backend version is allowed.
